隱藏鍵盤設定

Lazy

2019-09-22

就是這個選單確實是存在 m_inputSwitcherItems 裡面

如果你直接幹掉例如第五個東西，會幹掉左右功能

如果你直接幹掉第一個，會幹掉鍵盤設定功能

但是這時候有一個問題

你記得橫向的時候我們是沒有左右選單的嗎？

這樣你變成還要判定是否橫向，再決定是否幹掉

然後有些 app 不支援大手機尺寸，或者，如果你在 iPhone SE 裡面，是不支援左右鍵盤的

這樣貿然幹掉會出問題，所以這些都要考慮清楚

  • (BOOL)_isHandBiasSwitchVisible

可以 100% 確保不會出現以上這些問題，所以他相對來說是一個比較好的選擇

可以少寫很多判斷特殊狀況的 code

那我再說明為什麼不可以直接幹掉第一項鍵盤設定

因為這個鍵盤設定呢，他在 設定.app 裡面不會出現

所以你硬幹的話會導致第一個輸入法直接消失

所以他還是會 removeObjectAtIndex:0 沒錯，但是要先判斷是否第一個為鍵盤設定

ichitaso 的作法是，檢查 nsbundle 是否為 com.apple.preferences

這樣可以，但是我想讓你學去呼叫 private API，不是直接 hook 掉

所以你在 m_inputSwitcherItems 會看到裡面的項目，你要去判定該項目是否為鍵盤設定

是的話再 remove 他

這個時候你會需要呼叫一個 private Class 裡面的 private method

然後 private Class 你要自己宣告，例如

這樣才可以編譯

以上說明完畢

所以重點，你要看 m_inputSwitcherItems 什麼時候會被呼叫，這樣才會拿到正確的 m_inputSwitcherItems array 內容

假設我們有一個 UITableView

裡面的 cell 不是 UITableViewCell 而是一個 UITableViewPrivateCell 好了，我隨便取名的

這個 UITableViewPrivateCell 存在 UIKit 裡面

但是他不是一個公開的 class

那我們是可以使用他的

只是我們沒有 header

所以就

@interface UITableViewPrivateCell : NSObject
@end

讓程式碼認識到說，喔其實有這個 class 存在

這樣就可以用了

補充一點：
_isHandBiasSwitchVisible 你 hopper 看一下，會發現他叫了 [[UIKeyboardImpl sharedInstance] allowsReachableKeyboard]

allowsReachableKeyboard 再點進去看，你會發現他幫你做好檢查手機尺寸跟是不是橫向了

所以 Apple 幫我們做好了就不必再自己手寫判定

所以這題重點：

  1. 找到正確呼叫 ivar 的點
  2. 找到正確移除 objectIndexAt 0 的方式（不要暴力移除）
  3. 第二點跟 private class 有關

喔再補充一點，我們之前都是直接改 return 值或者參數傳遞值居多

但你應該也看得出來，現在在 hook 的時候裡面是要寫 code 的
所以要注意 %orig 在 hook function 的位置

先做後作會有差

然後，雖然叫你只看 UIInputSwitcherView，但這裡的意思是 %hook UIInputSwitcherView ，找裡面的 method 來hook

當你遇到 private class 的時候，就要去看 private class 本身的 header 或 hopper

所以應該總共會看兩個 class，UIInputSwitcherView 跟未知的 private class
